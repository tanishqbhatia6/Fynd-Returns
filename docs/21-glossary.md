# Glossary

All domain-specific terms used throughout ReturnProMax, listed alphabetically. This reference is intended for developers, QA engineers, and support staff who need precise definitions of the vocabulary used in the codebase, API, and admin UI.

---

## A

**Affiliate Order ID**
A unique order identifier assigned by Fynd when a Shopify order is linked as an "affiliate" order. Stored in `ReturnCase.fyndOrderId`. Not the same as the Shopify order name (`#1001`) or the internal Shopify GID. Used as the primary key for all Fynd return API calls.

**API Key**
A bearer token in the format `rpm_` followed by 40 hex characters (160-bit entropy). Grants programmatic access to the ReturnProMax external REST API. The full key is shown exactly once at creation time; only a bcrypt hash and an 8-character prefix are persisted. Managed via the `ApiKey` model. Transmitted in the `X-API-Key` HTTP header.

**Auto-Approve**
A per-shop setting (`ShopSettings.autoApproveEnabled`) that, when enabled, immediately sets a newly submitted return to `approved` status without manual admin review. Advanced rule-based auto-approve is configured via `autoApproveRulesJson`, allowing conditions on reason codes, product tags, order value, and more.

**AWB (Air Waybill)**
A logistics tracking number assigned by a carrier. ReturnProMax stores two AWB values per return case: `forwardAwb` (original shipment to customer) and `returnAwb` (reverse shipment back to warehouse). Populated from Fynd webhook payloads.

## B

**Blocklist**
A per-shop list of blocked customers or orders. Entries are stored in the `BlocklistEntry` model and matched by `type` (`email`, `phone`, `order_name`, or `ip`). When `ShopSettings.blocklistEnabled` is `true`, return submissions from blocked identifiers are rejected at the portal level.

**Bonus Credit**
An optional incentive (`ShopSettings.bonusCreditEnabled`) that adds an extra percentage (`bonusCreditPct`, default 10%) on top of the refund amount when the customer chooses store credit or exchange as their resolution type. The calculated bonus is stored in `ReturnCase.bonusCreditAmount`.

## C

**CUID (Collision-resistant Unique Identifier)**
The primary key format used for most database models. Generated by the `cuid()` function in Prisma, producing compact, URL-safe, monotonically sortable identifiers (e.g., `clxyz1234abcde`). Used for `Shop.id`, `ReturnCase.id`, `ReturnItem.id`, and others.

**Condition (Item Condition)**
A classification of the physical state of a returned item. Stored in `ReturnItem.condition`. Allowed values: `unused`, `used_good`, `used_damaged`, `defective`.

**Currency**
The ISO 4217 currency code of the order (e.g., `INR`, `USD`, `EUR`). Stored in `ReturnCase.currency` and auto-detected from the Shopify order at return creation time.

## D

**Discount Code Refund**
An alternative refund method (`ShopSettings.discountCodeRefundEnabled`) that issues a single-use Shopify discount code instead of a monetary refund. The code is prefixed with `discountCodePrefix` (default `RETURN`) and expires after `discountCodeExpiryDays` (default 90). The generated code and its value are stored in `ReturnCase.discountCode` and `ReturnCase.discountCodeValue`.

## E

**Exchange Order**
A new Shopify order created when the customer's resolution type is `exchange`. The replacement order's Shopify ID and name are stored in `ReturnCase.exchangeOrderId` and `ReturnCase.exchangeOrderName`. The items selected for exchange are persisted in `exchangeItemsJson`.

**Exchange Preference**
Free-text input from the customer describing their desired exchange (e.g., "Size L instead of M, same color"). Stored in `ReturnCase.exchangePreference`.

## F

**Fynd**
A third-party order management and logistics platform integrated with ReturnProMax. Fynd handles reverse logistics (pickup scheduling, carrier assignment, AWB generation). Communication occurs via the Fynd Platform/Storefront API and inbound webhooks. Configuration lives in `ShopSettings.fyndApiType`, `fyndEnvironment`, `fyndCompanyId`, `fyndApplicationId`, and `fyndCredentials` (AES-256-GCM encrypted).

**Fynd Order Mapping**
A cache table (`FyndOrderMapping`) that maps a Shopify order name to its corresponding Fynd order ID and shipment ID, avoiding repeated API lookups. Keyed by `(shopId, shopifyOrderName)`.

**Fynd Sync Status**
Tracks whether a return case has been successfully synchronized to Fynd. Stored in `ReturnCase.fyndSyncStatus`. Values: `pending`, `synced`, `failed`, `retry_scheduled`, `processing`, `pending_consolidation`. Retry metadata is in `fyndSyncRetries`, `fyndSyncError`, and `fyndSyncNextRetry`.

**Fynd Webhook Log**
An audit record (`FyndWebhookLog`) of every inbound webhook received from Fynd. Captures shipment ID, order ID, status, action taken (e.g., `refund_in_progress`, `status_updated`, `ignored`, `duplicate_ignored`), carrier/AWB details, and the full raw payload.

## G

**Green Return**
A sustainability feature (`ShopSettings.greenReturnsEnabled`) where the customer keeps the item and still receives a refund, avoiding reverse shipping costs and environmental impact. Applies to items below `greenReturnsThreshold` in value or tagged with specific product tags (`greenReturnsProductTags`). Flagged on the return case via `ReturnCase.isGreenReturn`.

## H

**HMAC (Hash-based Message Authentication Code)**
Used in two contexts: (1) Shopify uses HMAC-SHA256 to sign webhook payloads so the app can verify authenticity. (2) ReturnProMax signs outbound webhook payloads with HMAC-SHA256 using a per-subscription secret, placing the signature in the `X-RPM-Signature` header with a `sha256=` prefix.

## I

**Initiated**
An alias for the `pending` status used in some portal flows. Functionally equivalent to `pending` -- the return has been submitted but not yet reviewed.

## L

**Line Item**
A single product line within a Shopify order. Each line item that the customer wants to return becomes a `ReturnItem` record linked to the `ReturnCase`. Key fields: `shopifyLineItemId`, `title`, `variantTitle`, `sku`, `price`, `qty`.

**Lookup Session**
A temporary record (`LookupSession`) created when a customer searches for their order on the portal. Stores the hashed lookup value, matched return IDs, OTP state, and a short-lived JWT (`portalToken`). Sessions expire after a configured TTL and are periodically cleaned up.

## M

**Minimum Return Price**
A per-shop threshold (`ShopSettings.minimumReturnPrice`) below which items are not eligible for return. Stored as `Decimal(12,2)`.

## N

**Notification Log**
An audit record (`NotificationLog`) of every notification sent by the system. Tracks `channel` (`email`, `whatsapp`, `sms`), `recipient`, `eventType` (`new_return`, `approved`, `rejected`, `refunded`, `otp`, `custom_note`), and delivery status (`sent` or `failed`).

**No-Return Period**
A configurable blackout window (`ShopSettings.noReturnPeriodEnabled`, `noReturnPeriodStart`, `noReturnPeriodEnd`) during which all return submissions are blocked (e.g., during a major sale event).

## O

**OTP (One-Time Password)**
A verification code sent to the customer's email before granting portal access. Controlled by `ShopSettings.portalOtpEmailEnabled`; the `portalOtpSmsEnabled` schema field is locked on, but phone OTP starts fail closed until a real SMS/WhatsApp OTP delivery path is implemented. OTP state (sent timestamp, verification timestamp, attempt count) is tracked on the `LookupSession` record.

## P

**Portal**
The customer-facing return submission interface, embedded on the merchant's Shopify storefront. Customers look up orders by order number, email, phone, or return request number. The portal runs on the store domain and communicates with the app via CORS-enabled API endpoints.

**Portal Theme**
Customizable visual styling for the customer portal. Configured via `ShopSettings.portalThemeJson` and `portalConfigJson`. Includes brand logo (`brandLogoUrl`) and favicon (`brandFaviconUrl`), which may be Base64 data URIs or HTTPS URLs.

**Product Policies**
Per-product or per-category return policies (`ShopSettings.productPoliciesJson`). Each rule matches products by tags, collections, or product types and overrides the default return window and policy text.

## R

**Reason Code**
A machine-readable identifier for why the customer is returning an item (e.g., `wrong_size`, `defective`, `not_as_described`). Stored in `ReturnItem.reasonCode`. The available reasons are configured per shop in `ShopSettings.returnReasonsJson` and optionally per category in `returnReasonsByCategoryJson`.

**Refund Method**
How the refund is issued. Controlled by `ShopSettings.refundPaymentMethod`. Values: `original` (refund to original payment method), `store_credit` (Shopify store credit), `both` (split between original and store credit based on `refundStoreCreditPct`). Separate configurations exist for prepaid (`refundMethodPrepaidJson`) and COD (`refundMethodCODJson`) orders.

**Refund Status**
Tracks the refund lifecycle on a return case (`ReturnCase.refundStatus`). Common values: `null` (no refund attempted), `refunded`, `refund_in_progress`, `failed`.

**Resolution Type**
The customer's chosen outcome for the return. Stored in `ReturnCase.resolutionType`. Values: `refund` (default), `exchange`, `store_credit`, `replacement`.

**Restock Location**
The Shopify inventory location where returned items are restocked. Controlled by `ShopSettings.refundLocationMode`: `auto` uses the original fulfillment location, `manual` lets the admin choose. A fallback location ID can be set in `refundLocationId`.

**Restricted Product Tags**
Product tags (`ShopSettings.restrictedProductTagsJson`) that make items ineligible for return (e.g., `final-sale`, `non-returnable`).

**Restricted Regions**
Geographic regions (`ShopSettings.restrictedRegionsJson`) from which returns are not accepted.

**Return Case**
The central domain entity (`ReturnCase` model) representing a single return request. Contains customer info, order references, status, resolution details, Fynd integration state, and links to child `ReturnItem` and `ReturnEvent` records.

**Return Event**
An immutable audit log entry (`ReturnEvent` model) recording every action taken on a return case. Each event has a `source` (e.g., `admin`, `customer`, `system`, `fynd`), an `eventType` (e.g., `initiated`, `auto_approved`, `status_updated`, `refund_issued`, `note_added`), and an optional JSON payload.

**Return Fee**
An optional deduction (`ShopSettings.returnFeeAmount`) subtracted from the refund amount, typically to cover return shipping costs. Stored as `Decimal(12,2)` with an associated `returnFeeCurrency`.

**Return Item**
A single product being returned (`ReturnItem` model). Linked to a `ReturnCase`. Stores the Shopify line item ID, product details (title, variant, SKU, price, image), quantity, reason code, condition, and optional Fynd bag/shipment references.

**Return Label**
A shipping label for the customer to use when sending back items. Can be a URL (`ReturnCase.returnLabelUrl`) or structured data (`returnLabelJson`) containing carrier name, tracking number, label URL, and QR code URL.

**Return Offers**
Incentives offered to customers to keep items instead of returning them (`ShopSettings.returnOffersEnabled`, `returnOffersJson`). Offers can be percentage or flat discounts, matched by reason code or product tag.

**Return Request Number**
A human-readable identifier in the format `RPM-XXXXXXXX` (e.g., `RPM-A1B2C3D4`). Generated from the last 8 characters of the CUID, uppercased. Stored in `ReturnCase.returnRequestNo`. Used in customer-facing communications and portal lookups.

**Return Window**
The number of days after order fulfillment during which returns are accepted. Default: 30 days (`ShopSettings.returnWindowDays`). Can be overridden per product category via `productPoliciesJson`.

## S

**Session**
The Shopify OAuth session record (`Session` model). Managed by `@shopify/shopify-app-session-storage-prisma`. Stores access tokens, user details, scopes, and expiry. Supports both online (user-scoped) and offline (shop-scoped) sessions.

**Shop**
The top-level tenant entity (`Shop` model). Identified by `shopDomain` (e.g., `my-store.myshopify.com`). All return cases, settings, API keys, and webhook subscriptions belong to a shop.

**Shopify Admin GraphQL**
The GraphQL API used to interact with the Shopify platform. ReturnProMax uses it for order lookups, refund creation, discount code generation, metafield management, and store credit issuance. API version: `January26`.

**Split Refund**
A refund divided between the original payment method and store credit. Configured via `ShopSettings.refundPaymentMethod = "both"` with `refundStoreCreditPct` controlling the split percentage.

**Store Credit**
A refund issued as Shopify store credit rather than returned to the original payment instrument. One of the available refund methods. Can be combined with bonus credit for an additional incentive.

## W

**Webhook Subscription**
A registered endpoint (`WebhookSubscription` model) that receives outbound event notifications from ReturnProMax. Each subscription specifies a URL, a list of event types (e.g., `return.created`, `return.approved`, `return.rejected`, `return.refunded`, `return.status_changed`), and an HMAC secret for signature verification.

**WhatsApp Notifications**
Optional customer notifications via WhatsApp (`ShopSettings.whatsappEnabled`). Supports multiple providers: `meta_cloud`, `twilio`, `wati`, `interakt`. Configured with provider-specific API keys, phone number IDs, and sender numbers.
