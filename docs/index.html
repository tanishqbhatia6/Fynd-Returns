8 Settings Modules

10 configuration modules with full error handling, try/catch on all DB operations, and proper error/success feedback UI.

📋

Policy Rules

Return reasons, per-category reasons, restricted regions, minimum price, return offers. /app/settings/rules

⚙

Return Settings

Return window, fees, no-return periods (controlled toggle + date validation), photo requirements, auto-approve/refund, refund methods, discount codes. /app/settings/return-settings

🏷

Product Policies

Per-product return rules by tags, type, or collection. Custom windows, returnable toggle. First-match evaluation. /app/settings/product-policies

🚫

Customer Blocklist

Block by email, phone, order name, IP. Duplicate detection via unique constraint. Normalized values. /app/settings/blocklist

✅

Auto-Approve Rules

Conditional rules on order value, reason, product tag, customer return count. Approve or flag actions. /app/settings/auto-rules

🔗

Fynd Integration

Platform API credentials, environment (UAT/Prod), connection testing with debug log, password-masked secret input. /app/settings/integrations

🔔

Notifications

SMTP config with connection testing, 4 email events, custom templates with variable interpolation and live preview. /app/settings/notifications

🎨

Portal Appearance

Theme (colors, fonts, radius), tab visibility, language selection (15 langs), label overrides. /app/settings/widget

🔒

Permissions

read_all_orders scope management with OAuth status display. /app/settings/permissions

🧙

Setup Wizard

Step-by-step Fynd configuration — credentials, webhook registration, connection test, verification. /app/settings/setup

11 Security

Enterprise-grade security across all layers.

📱
