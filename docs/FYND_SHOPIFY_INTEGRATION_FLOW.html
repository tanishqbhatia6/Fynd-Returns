! Gaps & Risks Identified
Critical
No Shopify order/fulfillment webhooks
The app doesn't subscribe to Shopify's orders/fulfilled or fulfillments/create webhooks. It cannot proactively know when an order is shipped/delivered. Fulfillment status is checked on-demand only when a customer tries to create a return. Impact: Can't send proactive "your order is delivered, need to return?" notifications.
Critical
No background sync or retry queue
If a Fynd sync fails during return creation, there is no automatic retry. Admin must manually click "Retry Fynd sync". Failed webhooks are logged but not retried. No dead-letter queue exists. Impact: Returns can get stuck in a half-synced state indefinitely.
High
Webhook signature verification is optional
The FYND_WEBHOOK_SECRET env var is optional. If not set, any POST to /api/webhooks/fynd is accepted without verification. Impact: An attacker could forge refund webhooks and trigger unauthorized Shopify refunds.
High
Order-to-Fynd mapping is fragile
The app tries up to 5 different search strategies to match a Shopify order to Fynd (external_order_id, stripped prefix, channel_order_id, affiliate_order_id, order_id). If none match, tracking data is completely absent. Impact: Some orders may never show tracking data on the portal.
Medium
No expired session cleanup
cleanupExpiredSessions() is defined in portal-auth.server.ts but never called. Expired OTP sessions accumulate in the database. Impact: Database bloat over time.
Medium
No server-side Fynd status polling
Between webhook events, return status is stale. The portal uses 60s client-side auto-refresh, but the admin panel only updates when manually refreshed. No server-side cron to poll Fynd for status changes. Impact: Admin sees stale data; delays in processing returns.
Low
Product tags not passed for eligibility
ORDERS_BY_NAME_QUERY fetches variant.product.tags but fetchOrderByOrderNumber() doesn't include them in the mapped response. Restricted product tag rules may not work correctly for the Create Return flow. Impact: Non-returnable products might slip through.
